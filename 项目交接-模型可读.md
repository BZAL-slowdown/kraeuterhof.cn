## 4. 目前的核心业务结构

### 4.1 商品模块
- 商品模块实际目录是 `App/Cp`。
- 当前商品表逻辑围绕 `cp` 模块展开。
- 商品价格字段预期应该是：
  - 数据表列：`dr_1_cp.price` 或对应站点表的 `price`
  - 字段表记录：`dr_field` 中 `fieldname=price`、`relatedname=module`、`relatedid=cp 模块 ID`
- `Shop.php` 中的 `price_fields` 目前是：
  - `price`
  - `jiage`
  - `shoujia`
  - `sale_price`

### 4.2 订单逻辑
- `dayrui/App/Shop/Controllers/Home.php`
  - `create()` 会读取商品记录
  - `resolvePrice()` 会按 `price_fields` 寻找价格
  - 订单金额写入 `unit_price / total_amount / pay_amount`
- 当前代码逻辑已不再使用固定测试价作为最终生产逻辑。

### 4.3 支付配置逻辑
- 微信 H5 与支付宝 H5 都是通过后台配置写入 `dayrui/App/Shop/Config/Pay.php`。
- 证书/密钥文件位置要放在服务端可读但不要暴露公网下载的位置。
- 常见路径约定：
  - `cache/pay/wechat/apiclient_key.pem`
  - `cache/pay/wechat/wechatpay_public.pem`
  - `cache/pay/alipay/app_private_key.pem`
  - `cache/pay/alipay/alipay_public_key.pem`

## 6. 还没彻底解决的问题

### 6.1 后台产品编辑页没有出现“价格”字段
这是当前最关键的未解决项。

现象：
- 前台可以读到价格逻辑
- 但后台产品编辑页面里仍然看不到 `价格` 输入框

已经尝试过：
- 在 `dayrui/App/Module/Config/Update.php` 里自动插入字段
- 给 `cp` 模块补 `price` 列
- 给 `dr_field` 写入 `price` 字段定义
- 给模块后台列表配置加入 `price`
- 增加 `fix_cp_price.php` 作为手工兜底脚本

高概率原因：
- 模块缓存还没刷新
- `cp` 模块字段表没有真正更新到当前站点
- 站点实际模块目录大小写或站点 ID 与预期不一致

建议下一步：
1. 在宝塔终端执行：
   ```bash
   cd /www/wwwroot/kraeuterhof
   php fix_cp_price.php
   ```
2. 再到后台执行一次：
   - `系统` -> `系统维护` -> `系统缓存`
   - `更新缓存 / 一键更新缓存`
3. 再回到产品编辑页检查 `价格` 字段是否出现
4. 若仍无显示，直接查数据库：
   - `dr_module` 是否存在 `dirname=cp`
   - `dr_field` 是否存在 `fieldname=price`
   - `dr_1_cp`（或实际站点表）是否存在 `price` 列

### 6.2 短信验证码 / 邮件验证码仍需验证
之前出现过：
- 注册发送验证码失败
- 找回密码发送失败
- 邮件提示 SMTP 未配置

这通常不是前端模板问题，而是后台配置、接口、模板变量或第三方通道问题。

需要继续确认：
- 短信服务是否真的可发
- 邮件 SMTP 是否写入正确
- 注册默认用户组是否配置
- 开发者模式是否影响了跳转与提示

### 6.3 登录异常 / 系统错误
之前曾出现：
- 登录时弹 `系统错误`
- 明明账号密码正确仍无法进入

需要继续确认：
- 是会员账号校验问题
- 还是登录跳转/back 参数问题
- 还是后台缓存、默认组、模板路径不一致

### 6.4 备案信息与客户正式信息最终核对
页脚里备案号、公司名、域名、联系信息都要和客户备案保持一致。
这项还需要再核对一次实际模板输出，避免测试值混进正式站。
